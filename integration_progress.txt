INTEGRATION PATTERNS USED
================================================================================
  Settings Pages:
    - Store complex config as JSON in SystemSetting via key-value API
    - Load: getSystemSettings() -> find by key -> JSON.parse(value)
    - Save: createSystemSetting/updateSystemSetting with JSON.stringify
    - Track settingId in local state for create vs update logic

  Entity Pages:
    - Fetch from dedicated API endpoints
    - Map API response to local interface using adapter functions
    - Status mapping (e.g., API 'approved' -> UI 'published')
    - Async handlers with try/catch + toast feedback

  Common Fixes Applied:
    - toast import: 'sonner@2.0.3' -> 'sonner'
    - db-extended imports replaced with API calls
    - All mock arrays removed, replaced with useState([]) + useEffect
